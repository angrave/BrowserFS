    Information about a failed call to the BrowserFS API.

    Special thanks to Dropbox-JS for some of the error names/descriptions.

    See also:

    • https://raw.github.com/dropbox/dropbox-js/master/src/api_error.coffee

    Constant Summary

    NETWORK_ERROR =

    Status value indicating an error at the XMLHttpRequest layer.

    This indicates a network transmission error on modern browsers. Internet Explorer might cause this code to be reported on some API server errors.

    0
    INVALID_PARAM =

    Status value indicating an invalid input parameter.

    400
    INVALID_TOKEN =

    Status value indicating an expired or invalid OAuth token.

    The OAuth token used for the request will never become valid again, so the user should be re-authenticated.

    401
    AUTH_ERROR =

    Status value indicating an authentication error of some sort.

    403
    NOT_FOUND =

    Status value indicating that a file or path was not found in the filesystem.

    This happens when trying to read from a non-existing file, readdir a non-existing directory, write a file into a non-existing directory, etc.

    404
    DRIVE_FULL =

    Status value indicating that the filesystem is full to capacity.

    507
    NOT_SUPPORTED =

    Indicates that the given method is not supported on the current filesystem.

    405
    PERMISSIONS_ERROR =

    Indicates that you lack sufficient permissions to perform the indicated task. This could be due to a filemode error.

    900
