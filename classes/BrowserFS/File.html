    Class: BrowserFS.File

    Defined in: src/core/file.coffee

    Overview

    Base class that contains the interface for a file object. BrowserFS uses these as a replacement for numeric file descriptors.

    - (void) write(buffer, offset, length, position, cb)

    Core: Write buffer to the file. Note that it is unsafe to use fs.write multiple times on the same file without waiting for the callback. the file.

    Parameters:

    • buffer (BrowserFS.node.Buffer) — Buffer containing the data to write to
    • offset (Number) — Offset in the buffer to start reading data from.
    • length (Number) — The amount of bytes to write to the file.
    • position (Number) — Offset from the beginning of the file where this data should be written. If position is null, the data will be written at the current position.
    • cb (Function(BrowserFS.ApiError, Number, BrowserFS.node.Buffer)) — The number specifies the number of bytes written into the file.

    - (void) read(buffer, offset, length, position, cb)

    Core: Read data from the file.

    Parameters:

    • buffer (BrowserFS.node.Buffer) — The buffer that the data will be written to.
    • offset (Number) — The offset within the buffer where writing will start.
    • length (Number) — An integer specifying the number of bytes to read.
    • position (Number) — An integer specifying where to begin reading from in the file. If position is null, data will be read from the current file position.
    • cb (Function(BrowserFS.ApiError, Number, BrowserFS.node.Buffer)) — The number is the number of bytes read
